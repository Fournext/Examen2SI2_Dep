Para diseñar una base de datos que maneje materias, temas, niveles (grados) y maestros, puedes seguir un enfoque relacional clásico. Aquí te dejo un modelo lógico básico con entidades, relaciones y claves primarias/foráneas:

🧩 ENTIDADES Y ATRIBUTOS

1. Maestro

  • id_maestro (PK)

  • nombre

  • apellido

  • correo

  • telefono

2. Nivel

  • id_nivel (PK)

  • nombre_nivel (ej. "Primero", "Segundo", "Tercero")

3. Materia

  • id_materia (PK)

  • nombre_materia

  • id_nivel (FK → Nivel)

4. Tema

  • id_tema (PK)

  • titulo_tema

  • descripcion

  • id_materia (FK → Materia)

5. MaestroMateria

(relación N:N entre Maestro y Materia)

  • id_maestro (FK → Maestro)

  • id_materia (FK → Materia)

  • anio_academico

🔗 RELACIONES

  • Un maestro puede impartir varias materias, y una materia puede ser impartida por varios maestros → relación N:N, implementada mediante la tabla intermedia MaestroMateria.

  • Una materia pertenece a un único nivel, pero un nivel puede tener muchas materias.

  • Una materia puede tener varios temas, pero un tema pertenece a una única materia.

📊 DIAGRAMA ENTIDAD RELACIÓN SIMPLIFICADO

lua
Maestro --< MaestroMateria >-- Materia --< Tema

                       v
                     Nivel

🧱 EJEMPLO DE SQL

sql
CREATE TABLE Nivel (
    id_nivel SERIAL PRIMARY KEY,
    nombre_nivel VARCHAR(50) NOT NULL
);

CREATE TABLE Maestro (
    id_maestro SERIAL PRIMARY KEY,
    nombre VARCHAR(100),
    apellido VARCHAR(100),
    correo VARCHAR(100),
    telefono VARCHAR(20)
);

CREATE TABLE Materia (
    id_materia SERIAL PRIMARY KEY,
    nombre_materia VARCHAR(100),
    id_nivel INTEGER REFERENCES Nivel(id_nivel)
);

CREATE TABLE Tema (
    id_tema SERIAL PRIMARY KEY,
    titulo_tema VARCHAR(100),
    descripcion TEXT,
    id_materia INTEGER REFERENCES Materia(id_materia)
);

CREATE TABLE MaestroMateria (
    id_maestro INTEGER REFERENCES Maestro(id_maestro),
    id_materia INTEGER REFERENCES Materia(id_materia),
    anio_academico VARCHAR(10),
    PRIMARY KEY(id_maestro, id_materia)
);

🧩 ENTIDADES PRINCIPALES Y RELACIONES

1. Alumno

  • id_alumno (PK)

  • nombre

  • apellido

  • fecha_nacimiento

  • correo

  • telefono

2. Profesor

  • id_profesor (PK)

  • nombre

  • apellido

  • correo

  • especialidad

3. Nivel o Grado

  • id_nivel (PK)

  • nombre_nivel (ej. "Primero Secundaria")

4. Materia

  • id_materia (PK)

  • nombre_materia

  • id_nivel (FK → Nivel)

5. Gestion

  • id_gestion (PK)

  • anio (ej. 2025)

  • periodo (opcional, por ejemplo "1er semestre", "2do semestre")

6. Curso

Relaciona una materia con un profesor en un año académico.

  • id_curso (PK)

  • id_materia (FK → Materia)

  • id_profesor (FK → Profesor)

  • id_gestion (FK → Gestion)

7. Inscripcion

Relaciona a un alumno con un curso.

  • id_inscripcion (PK)

  • id_alumno (FK → Alumno)

  • id_curso (FK → Curso)

  • fecha_inscripcion

8. Calificacion

  • id_calificacion (PK)

  • id_inscripcion (FK → Inscripcion)

  • nota

  • fecha

  • tipo (ej. Parcial, Final, Trabajo)

🔗 RELACIONES CLAVE

  • Un profesor dicta muchas materias en diferentes gestiones.

  • Un alumno se inscribe a varias materias (cursos) por gestión.

  • Cada materia se dicta en un cierto nivel (grado).

  • Cada inscripción puede tener múltiples calificaciones.

🧱 SQL BÁSICO (ESQUEMA)

sql
CREATE TABLE Alumno (
    id_alumno SERIAL PRIMARY KEY,
    nombre VARCHAR(100),
    apellido VARCHAR(100),
    fecha_nacimiento DATE,
    correo VARCHAR(100),
    telefono VARCHAR(20)
);

CREATE TABLE Profesor (
    id_profesor SERIAL PRIMARY KEY,
    nombre VARCHAR(100),
    apellido VARCHAR(100),
    correo VARCHAR(100),
    especialidad VARCHAR(100)
);

CREATE TABLE Nivel (
    id_nivel SERIAL PRIMARY KEY,
    nombre_nivel VARCHAR(100)
);

CREATE TABLE Materia (
    id_materia SERIAL PRIMARY KEY,
    nombre_materia VARCHAR(100),
    id_nivel INTEGER REFERENCES Nivel(id_nivel)
);

CREATE TABLE Gestion (
    id_gestion SERIAL PRIMARY KEY,
    anio INTEGER,
    periodo VARCHAR(50)
);

CREATE TABLE Curso (
    id_curso SERIAL PRIMARY KEY,
    id_materia INTEGER REFERENCES Materia(id_materia),
    id_profesor INTEGER REFERENCES Profesor(id_profesor),
    id_gestion INTEGER REFERENCES Gestion(id_gestion)
);

CREATE TABLE Inscripcion (
    id_inscripcion SERIAL PRIMARY KEY,
    id_alumno INTEGER REFERENCES Alumno(id_alumno),
    id_curso INTEGER REFERENCES Curso(id_curso),
    fecha_inscripcion DATE
);

CREATE TABLE Calificacion (
    id_calificacion SERIAL PRIMARY KEY,
    id_inscripcion INTEGER REFERENCES Inscripcion(id_inscripcion),
    nota NUMERIC(5,2),
    fecha DATE,
    tipo VARCHAR(50)
);
